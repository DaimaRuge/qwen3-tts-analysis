## 项目概述

### 核心目标
在中国开发团队基于国际领先的AI大模型API，构建面向北美市场的Agent硬件系统与产品。

### 技术栈
- **核心AI引擎**：Google Gemini API、OpenAI GPT API、Claude API
- **开发工具**：Claude Code、GitHub Copilot
- **目标市场**：北美（美国、加拿大）

### 关键挑战
1. 国际AI服务的地域限制与支付障碍
2. 多账号管理与团队协作
3. 北美本地化测试需求

## 账号管理方案

### 安全最佳实践

1. **密码管理**
   - 使用密码管理器（1Password/Bitwarden）
   - 每个账号独立强密码
   - 定期更换密码（3个月周期）

2. **两步验证（2FA）**
   - 所有账号强制开启
   - 使用Authenticator App（非SMS优先）
   - 保存备份码到安全位置

3. **访问控制**
   - 主管理员账号唯一
   - 分级权限管理
   - 操作日志记录

## 风险与合规建议

### 1. 服务地域限制

**风险**：部分AI服务对中国IP有限制

**缓解措施**：
- 使用稳定的商业VPN/代理
- 北美手机卡用于验证
- 避免频繁切换IP

### 2. 支付问题

**风险**：国内信用卡可能无法支付

**解决方案**：
- 使用北美同事的美国信用卡
- 绑定PayPal（美国区）
- 购买礼品卡充值（OpenAI/Google Play）

### 3. 账号安全

**风险**：多账号管理复杂，易被封禁

**建议**：
- 每个账号使用独立IP注册
- 避免共享账号登录
- 遵守各平台使用条款
- 定期登录保持活跃

### 4. 合规建议

- 注册账号时使用真实信息
- 遵守美国出口管制规定
- API调用不用于禁止领域（军事、监控等）
- 保留订阅付款凭证
